# User Stories – Airbnb Clone Backend

This document contains the main user stories derived from the Use Case Diagram of the Airbnb Clone backend.

Each story is written from the user’s perspective and describes a key interaction with the system.

---

## 1. Guest User Stories
1. As a guest, I want to create an account so that I can book properties securely.
2. As a guest, I want to search and view properties so that I can find suitable accommodation.
3. As a guest, I want to make bookings so that I can reserve properties for my travel dates.
4. As a guest, I want to make secure online payments so that I can confirm my bookings instantly.
5. As a guest, I want to leave reviews and ratings so that I can share my experience with others.

---

## 2. Host User Stories
1. As a host, I want to create and manage property listings so that I can rent them out to guests.
2. As a host, I want to view and manage bookings for my properties so that I can plan availability.
3. As a host, I want to message guests so that I can coordinate check-in details easily.

---

## 3. Admin User Stories
1. As an admin, I want to manage users and properties so that I can maintain platform quality.
2. As an admin, I want to monitor bookings and payments so that I can ensure smooth operation.

---

## 4. Payment Gateway Story
1. As a payment system, I want to process transactions securely so that users can trust the platform.

---

These user stories form the foundation for designing API endpoints, validations, and backend logic.
